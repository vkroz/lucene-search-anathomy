Detailed Description

Provides a single Fields term index view over an IndexReader.

This is useful when you're interacting with an IndexReader implementation that consists of sequential sub-readers (eg DirectoryReader or MultiReader) and you must treat it as a LeafReader.

NOTE: for composite readers, you'll get better performance by gathering the sub readers using IndexReader#getContext() to get the atomic leaves and then operate per-LeafReader, instead of using this class.
