Detailed Description

An interface for implementations that support 2-phase commit.

You can use TwoPhaseCommitTool to execute a 2-phase commit algorithm over several TwoPhaseCommits.

.experimental

long org.apache.lucene.index.TwoPhaseCommit.commit ( ) throws IOException

The second phase of a 2-phase commit.

Implementations should ideally do very little work in this method (following prepareCommit(), and after it returns, the caller can assume that the changes were successfully committed to the underlying storage.

Implemented in org.apache.lucene.index.IndexWriter.

◆ prepareCommit()

long org.apache.lucene.index.TwoPhaseCommit.prepareCommit ( ) throws IOException

The first stage of a 2-phase commit.

Implementations should do as much work as possible in this method, but avoid actual committing changes. If the 2-phase commit fails, rollback() is called to discard all changes since last successful commit.

Implemented in org.apache.lucene.index.IndexWriter.

void org.apache.lucene.index.TwoPhaseCommit.rollback ( ) throws IOException

Discards any changes that have occurred since the last commit.

In a 2-phase commit algorithm, where one of the objects failed to commit() or prepareCommit(), this method is used to roll all other objects back to their previous state.

Implemented in org.apache.lucene.index.IndexWriter.
